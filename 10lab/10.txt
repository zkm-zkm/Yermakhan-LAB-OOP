4)class niNum(Exception):
Создаём собственный класс исключения niNum, который наследуется от стандартного класса Exception.

Это позволяет нам создавать специфические ошибки, связанные с нашим кодом.

python
Копировать код
    def __init__(self, number):
        super().__init__(f" Ошибка: число {number} отрицательное! Введите положительное число.")
Конструктор (__init__) нашего исключения.

super().__init__(...) вызывает конструктор базового класса Exception, чтобы передать туда сообщение об ошибке.

Сообщение формируется динамически: если число отрицательное, мы показываем его пользователю.

python
Копировать код
def check(number):
    if number < 0:
        raise niNum(number)
    else:
        print(f" Число {number} корректно!")
Функция check проверяет переданное число.

Если число отрицательное, вызывается исключение niNum(number) с соответствующим сообщением.

Если число положительное или ноль, выводится сообщение, что число корректно.

python
Копировать код
if __name__ == "__main__":
Проверяем, что скрипт запущен напрямую, а не импортирован как модуль.

python
Копировать код
    try:
        num = float(input("Введите число: "))
        check(num)
Запрашиваем число у пользователя и преобразуем его в float.

Передаём число в функцию check, которая может выбросить исключение, если число отрицательное.

python
Копировать код
    except ValueError:
        print(" Ошибка: введено не число.")
Ловим ошибку ValueError, если пользователь ввёл не число (например, текст).

python
Копировать код
    except niNum as e:
        print(e)
Ловим наше собственное исключение niNum.

as e позволяет получить объект исключения, а print(e) выводит сообщение, которое мы задали в конструкторе.

✅ Итог:
Программа:

Запрашивает число у пользователя.

Проверяет, что оно не отрицательное.

Если введено отрицательное число → выбрасывает собственное исключение с сообщением.

Если введено не число → сообщает об этом.

Если число корректное → подтверждает корректность.
